1. Web Accessibility (a11y) Kya Hai?

Accessibility ka matlab hai apni website ko aisa banana ke har koi (shamil disabled log bhi) use kar sake — jaise woh log jo screen readers use karte hain (blind users).

Accessibility Ke 3 Golden Rules:

  • 🖼️ Images: Hamesha alt attribute dein. Screen readers image ki jagah alt text padhte hain.
  • 🏷️ Forms: Har <input> ke saath <label for="..."> laazmi hona chahiye.
  • 🏗️ Structure: Headings (h1 se h6) ko sequence mein use karein, aur Semantic tags (main, nav, footer) use karein.
<!-- ❌ Galat (Not Accessible) -->
<img src="logo.png">
<input type="text" placeholder="Naam likhein">

<!-- ✅ Sahi (Accessible) -->
<img src="logo.png" alt="Code Academy Nowshera Logo">

<label for="username">Naam:</label>
<input type="text" id="username" placeholder="Naam likhein">

2. ARIA Roles (WAI-ARIA)

Jab normal HTML tags kafi na hon (maslan koi complex custom widget ho), tab hum ARIA attributes (Accessible Rich Internet Applications) use karte hain taake screen readers ko samajh aye.

<!-- Agar div ko button ki tarah use kar rahe hain -->
<div role="button" aria-pressed="false" tabindex="0" onclick="submit()">
  Submit Form
</div>

<!-- Screen reader ko notify karne ke liye (Live updates) -->
<div aria-live="polite">
  Aapka message send ho gaya hai.
</div>

💡 Rule of Thumb: Pheli koshish hamesha native HTML tags (jaise <button>) use karne ki karein. Agar woh mumkin na ho toh ARIA use karein.

3. Clean Code Best Practices

PracticeDescription
IndentationNested tags ko properly indent (spaces/tabs) karein taake code padhna aasan ho.
LowercaseHTML tags aur attributes hamesha lowercase mein likhein. (e.g. <DIV> ki bajaye <div>).
QuotesAttribute values ko hamesha double quotes " " mein rakhein.
CommentsComplex sections ya layouts ke oopar <!-- comments --> zaroor likhein.
SeparationHTML mein sirf structure rakhein. Styling CSS mein aur logic JavaScript mein. (Inline styles avoid karein).
 
<!-- ❌ Messy Code -->
<DIV CLASS=card><H2>Title</H2><p style="color:red">text</p></DIV>

<!-- ✅ Clean Code -->
<!-- Main Card Section -->
<div class="card">
  <h2>Title</h2>
  <p class="text-danger">text</p>
</div>

4. Code Validation (W3C Validator)

Apna HTML code likhne ke baad usay hamesha W3C Validator par check karein ke koi error toh nahi.

  1. Website kholiye: validator.w3.org
  2. "Validate by Direct Input" tab select karein.
  3. Apna HTML code paste karein aur "Check" dabayein.
  4. Errors (jaise missing alt tags, unclosed tags) ko theek karein.

  • Task 3: Screen reader extension (e.g. ChromeVox) install karein aur aankhein band karke apni website ko keyboard (Tab key) se navigate karne ki koshish karein.
